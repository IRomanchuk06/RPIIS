# Лабораторная работа №1
## Условие задания (вариант 16)
Хеш-таблица. Вставка элемента в таблицу. Поиск элемента в таблице. Удаление элемента из таблицы.

**Цель:** Разработать программу для работы с хеш-таблицей, обеспечивающую эффективные операции вставки, поиска и удаления элементов.

**Задача:** Реализовать хеш-функцию. Создать алгоритм вставки элемента с учетом возможных коллизий, поиска и удаления элемента из хеш-таблицы. Провести тестирование программы на различных входных данных для проверки ее корректности и эффективности.

# Алгоритм 


## Общий алгоритм

1. **Основная функция**:
    - Объявление переменных: `arrSize`, `arr[100]`, `path`, `ch`.
    - Запрос пользователя на выбор файла и чтение ввода пользователя в переменную `ch`.
    - В зависимости от ввода пользователя (`ch`), установка переменной `path` в соответствующий файл.
    - Вызов функции `run_testcase(path, arrSize, arr)` для чтения данных из файла в массив `arr`.
    - Инициализация `arrSize` равным 10.
    - Объявление `Node** pTable`.
    - Вызов `create(arrSize)` для создания хеш-таблицы `pTable`.
    - Использование цикла для добавления каждого элемента из `arr` в хеш-таблицу.
    - Отображение меню пользователю и выполнение соответствующих действий в зависимости от ввода пользователя до тех пор, пока пользователь не выберет выход.

2. **Функция run_testcase**:
    - Открытие указанного файла `path` (arr1.txt, arr2.txt, arr3.txt).
    - Если файл не открыт, выводится сообщение об ошибке.
    - В противном случае, считывание первого целого числа из файла в переменную `arrSize`.
    - С использованием цикла считывание оставшихся целых чисел из файла в массив `arr`.
    - Закрытие файла.

3. **Функция create**:
    - Выделение памяти для хеш-таблицы (`Node** pTable`) размером `tableSize`.
    - Инициализация каждого элемента хеш-таблицы значением `NULL`.
    - Возврат указателя на созданную хеш-таблицу.

4. **Функция add**:
    - Создание нового узла `Node` и установка его значения `inf` во входное значение.
    - Расчет хеш-индекса `i` для входного значения.
    - Если соответствующий индекс в хеш-таблице пуст, вставка нового узла туда.
    - В противном случае, вставка нового узла в начало связанного списка по этому индексу.

5. **Функция search**:
    - Расчет хеш-индекса `i` для входного значения.
    - Обход связанного списка по индексу `i` и поиск входного значения.
    - Если найдено и не помечено как удаленное, возвращается индекс `i`.
    - Если не найдено или помечено как удаленное, возвращается -1.

6. **Функция remove**:
    - Расчет хеш-индекса `i` для входного значения.
    - Обход связанного списка по индексу `i` и пометка узла с входным значением как удаленного, если найдено.
